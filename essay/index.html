The Agent Loop Needs Plumbing, Not Snippets

Most "AI search" APIs are search engines wearing different clothes. A different category of product is emerging to fill the gap, and it deserves a name.

Kevin Nguyen  ·  May 2026  ·  ~5 min read
· · ·

Most "AI search" APIs today are search engines wearing different clothes. The endpoint is new, the SDK is new, the snippets are cleaner. The underlying model is the same one the web has used since 1998: a query goes in, a list of ranked text fragments comes back, a human decides what to do next.

That model breaks when the reader is an agent. Agents don't scan ten blue links. They consume facts, cite them, act on them, and have to answer for them. When the input is shaped for human triage and the output drives an automated workflow, the gap between the two is paid in hallucinations, brittle scrapers, and silent failures that look like model problems but are really data problems.

A different category of product is emerging to fill that gap. It deserves a name.

The Snippet Era Is Over

Call the current generation of AI search APIs snippet-era infrastructure. They wrap commodity search indexes, return cleaner excerpts, and hand the rest to the developer. They are better than scraping. They are not yet infrastructure.

The tell is in what they don't do. They don't report how confident they are in any individual fact. They don't distinguish the page that actually supports a claim from the page that merely contains the keywords. They don't structure their output by the schema the agent asked for — they structure it the way a human reading a results page would skim it. The cleanup happens after the API call, in code the developer maintains and re-debugs every time a website changes.

This is the same relationship application code had with payments before Stripe, or with email before Resend. Useful. Not yet load-bearing.

What Infrastructure Looks Like

If web infrastructure for agents is a real category, it has five primitives.

  • Perceive. Ground a question against the live web with citations attached. Not "top ten results" — the actual passages that support a claim.
  • Read. Convert any URL into clean, model-ready content. Strip the chrome, render the JavaScript, pull the PDF.
  • Reason. Run a multi-hop research task with verifiable basis — every output field traceable to the document and span that produced it.
  • Watch. Detect material change on a query, on a schedule, without standing up a cron-and-diff pipeline.
  • Source. Discover entities from a brief — not the URLs already known, the ones not yet known.

Each primitive is something an agent does on every meaningful task. Each one is something the snippet era leaves to the developer to assemble — usually by gluing together half a dozen tools, none of which know about each other, and none of which return their answers in a shape the next step can consume directly.

Confidence Is the Unit

Every infrastructure category in software has a boring primitive at its center that nobody asks for in user research but everyone ends up needing in production. For payments it was idempotency keys. For object storage it was eventual consistency guarantees. For databases it was the foreign-key constraint.

For web infrastructure for agents, that primitive is calibrated confidence per field. Not a single confidence score on the whole response. A specific, evaluated, comparable number on every individual fact, attached to the citation that produced it.

Confidence is what makes web data measurable, and measurable is what makes it shippable. With it, an agent can decide whether to escalate, re-query, or block its own action. Without it, every claim is the same claim, and every claim has to be either trusted blindly or audited manually. Most production teams pick neither and just lower their ambitions until the failures stop being noticeable.

Who Wins

The category winner will not be the team with the best snippet quality. Snippet quality is the price of entry. It will be the team that commits hardest to a typed, cited, confidence-scored, monitorable, sourceable surface as their actual product — and treats search as the on-ramp, not the destination.
